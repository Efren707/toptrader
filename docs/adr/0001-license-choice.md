# 0001 - Use the MIT License

- Status: Accepted
- Date: 2026-07-16

## Context

This is a public resume/portfolio project. The license needs to signal that the code is freely reusable/reviewable by anyone evaluating it (recruiters, other developers), without adding legal friction.

## Options considered

- **MIT** - maximally permissive, minimal boilerplate, most widely recognized.
- **Apache 2.0** - similar permissiveness, adds an explicit patent grant; more common in larger corporate-backed projects, more legal text.
- **No license (all rights reserved)** - default GitHub behavior if no LICENSE file is added; technically restricts others from reusing or even running the code.

## Decision

Use the MIT License.

## Consequences

- Anyone can view, copy, and reuse the code with attribution, which fits the goal of a public portfolio piece.
- No patent-grant language (not a concern for this project's scope).
